HarnessDeck Product Design / B + 1

Raycast / Linear 感的高密度开发者工具原型方向

这份 HTML 设计板用于描述 HarnessDeck 的第一轮静态视觉方向。三种方案都围绕菜单栏控制中心和二级工作台展开，重点看信息层级、导航方式、Profile 到 dry-run deploy plan 的主路径。

1

Command Deck：命令中心优先

这个方向把菜单栏面板设计成日常主入口，像 Raycast 一样让用户先搜索、再执行。Workbench 服务于更重的 Profile 和 Sync 管理，适合高频切换 Profile、账号和 target 的用户。

推荐场景：每天频繁在 Claude Code 和 Codex 间切换任务，用户更关心快速状态判断和命令入口。视觉重点是 command bar、状态密度和即时操作。

2

Sync Ops：同步治理优先

这个方向把 Workbench 设计成操作台，核心是 Profile 到 target 的部署与冲突治理。菜单栏更像运行摘要，真正的重点在 deploy plan、风险等级、manifest 和 rollback 线索。

推荐场景：用户担心配置被写坏，需要在每次同步前看清变化。视觉重点是表格、diff、风险标签和审计感。

3

Lifecycle Console：生命周期优先

这个方向把产品闭环直接放到界面上，Discover、Profile、Sync、Operate、Improve 形成一条清楚路径。它更适合展示最终目标态，也能让第一次使用的用户理解 HarnessDeck 负责哪条链路。

推荐场景：希望原型能讲清完整愿景，同时保留高密度工具感。视觉重点是生命周期阶段、当前进度、推荐动作和下一步。
